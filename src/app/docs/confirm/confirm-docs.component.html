The confirm service launches simple confirmation dialogs to allow users to confirm actions.

You selected the "{{ selectedText }}" button, which has an action of "{{ selectedAction }}."

You selected the "{{ selectedAction }}" action.

A confirmation dialog focuses user attention on a specific question and is appropriate for scenarios that meet the following criteria:

  • An action could be destructive or especially costly when performed by mistake.
  • Users must respond to a single question to proceed with the action.
  • The response to the question requires no input fields.

The question portion of the dialog must contain only one question.

  • The text can be more than one sentence.
  • Be concise and direct. Don't use this component if the context of the question requires significant explanation, illustration, or dynamic feedback of downstream consequences.
  • Avoid verbose or complicated phrasing. For example, only start questions with “Are you sure you want to” in rare cases where you need to emphasize that users should think twice before proceeding.

The answer portion of the dialog answers the question or tells users what will happen. It contains up to three options that are represented as terminal actions (buttons) with labels that clearly indicate the action that occurs when users select the buttons. For example, use "Delete" for the primary button text when the question asks users to confirm that they want to delete something, such as "Delete account?"

  • A primary button triggers the recommended or most-common action.
  • A secondary button triggers a less-common action.
  • A link button cancels the action.
  • Each button closes the dialog.
